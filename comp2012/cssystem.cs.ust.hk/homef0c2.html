Quick start on SSH/SCP/SFTP

  • On Unix
  • On Windows
  • On Web Browser

On UNIX:

Outgoing access: similar to telnet, just do a "ssh2 <hostname>"

Copy files around:

e.g. You are at hostA and wish to copy file abc.txt from directory /homes/test/papers at hostA to directory /project/mytest/papers at hostB:
At hostA:"scp2 /homes/test/papers/abc.txt hostB:/project/mytest/papers"
Now, if you are at hostB and want to do the same operation:
At hostB:"scp2 hostA:/homes/test/papers/abc.txt /project/mytest/papers"

SFTP:

Similar to regular command-line FTP, do a "sftp2 <hostname>"
You will then get a command prompt similar to FTP as:

sftp>

Basic commands like "ls, cd, get, mget, put, mput" are the same. To transfer binary file, you may type in:

sftp>binary


On Windows:

If you prefer the command-line mode, you may start a "Command Prompt" by "Start" -> "Programs" -> "Accessories" -> "Command Prompt"

At the prompt, you can use all the commands described above for UNIX.

If you prefer the GUI mode, you may start the "SSH Secure Shell Client". A copy can be downloaded here locally. This requires your CSD UNIX username/password.

After successful installation, there should be a "SSH Secure Shell Client" icon on your desktop, double click it to open the application and the terminal starts up.

Press "Enter", a dialog will show up and prompt you for hostname and username, input your desire hostname and your username into the corresponding boxes:

If this is the first time you connect to the remote UNIX workstation, a warning dialog will pop up and ask for your decision, you may simply click "Yes".

Then a little dialog, "Add Profile", will pop up on the upper-right-handed corner, you may store this session as a new profile so that you don't have to type in the username and hostname again, select a self-descriptive profile name, e.g. the hostname you are connecting to.

You're done. You are now getting a secure remote terminal, if you would like to do file-transfer, click on the icon circled in red as shown.

Another new window starts up and you may just drag-n-drop your files between your local machines and the remote UNIX workstation. The left-panel is your local machine file explorer while the right one is the remote side. The bottom panel is the transfer status for pending/active file transfers.


On Web browser:

If you are out of town or in front of a public computer terminal which do not have SSH client installed and you do not have the privilege to install one, but the machine does have Java runtime installed for the browser (with version J2SE 1.4+), you may try this inline SSH client through the web browser, both Netscape and Internet Explorer should work the same.

First, click this link. If the following dialog pops up, click "Yes".

Once the Java applet loaded completed, choose "New Connection".

At the Hostname field, specify the hostname you would like to connect to, e.g. ras.cse.ust.hk and type in your username.

When this dialog pops up, click "Yes".

When this dialog pops up, choose "password" and click "Proceed".

When prompted, type in your CSD UNIX password.

Now, you are having a SSH terminal open at the host you specified.

Once you are done, type in "exit" and when prompted to save the configuration, click "No".
